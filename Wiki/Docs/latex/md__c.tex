\hypertarget{md__c_autotoc_md91}{}\doxysubsection{Why yet another type checking module?}\label{md__c_autotoc_md91}
There are hundreds of type checking modules on npm, unfortunately, I couldn\textquotesingle{}t find any that fit my needs\+:


\begin{DoxyItemize}
\item Includes both type methods and ability to get the type
\item Types of primitives returned as lowercase and object types as camelcase
\item Covers all built-\/ins
\item Unsurprising behavior
\item Well-\/maintained
\item Comprehensive test suite
\end{DoxyItemize}

For the ones I found, pick 3 of these.

The most common mistakes I noticed in these modules was using {\ttfamily instanceof} for type checking, forgetting that functions are objects, and omitting {\ttfamily symbol} as a primitive.\hypertarget{md__c_autotoc_md92}{}\doxysection{Related}\label{md__c_autotoc_md92}
