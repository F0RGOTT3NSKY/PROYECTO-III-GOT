{\ttfamily Cache\+Policy} tells when responses can be reused from a cache, taking into account \href{http://httpwg.org/specs/rfc7234.html}{\texttt{ H\+T\+TP R\+FC 7234}} rules for user agents and shared caches. If {\ttfamily options.\+shared} is {\ttfamily true} (default), then the response is evaluated from a perspective of a shared cache (i.\+e. {\ttfamily private} is not cacheable and {\ttfamily s-\/maxage} is respected). If {\ttfamily options.\+shared} is {\ttfamily false}, then the response is evaluated from a perspective of a single-\/user cache (i.\+e. {\ttfamily private} is cacheable and {\ttfamily s-\/maxage} is ignored). {\ttfamily shared\+: true} is recommended for H\+T\+TP clients.

{\ttfamily options.\+cache\+Heuristic} is a fraction of response\textquotesingle{}s age that is used as a fallback cache duration. The default is 0.\+1 (10\%), e.\+g. if a file hasn\textquotesingle{}t been modified for 100 days, it\textquotesingle{}ll be cached for 100$\ast$0.1 = 10 days.

Per the R\+FC, the cache should take into account the time between server-\/supplied {\ttfamily Date} and the time it received the response. The R\+F\+C-\/mandated behavior creates two problems\+:


\begin{DoxyItemize}
\item Servers with incorrectly set timezone may add several hours to cache age (or more, if the clock is completely wrong).
\item Even reasonably correct clocks may be off by a couple of seconds, breaking {\ttfamily max-\/age=1} trick (which is useful for reverse proxies on high-\/traffic servers).
\end{DoxyItemize}

Previous versions of this library had an option to ignore the server date if it was \char`\"{}too inaccurate\char`\"{}. To support the {\ttfamily max-\/age=1} trick the library also has to ignore dates that pretty accurate. There\textquotesingle{}s no point of having an option to trust dates that are only a bit inaccurate, so this library won\textquotesingle{}t trust any server dates. {\ttfamily max-\/age} will be interpreted from the time the response has been received, not from when it has been sent. This will affect only \href{https://tools.ietf.org/html/rfc1149}{\texttt{ R\+FC 1149 networks}}.
