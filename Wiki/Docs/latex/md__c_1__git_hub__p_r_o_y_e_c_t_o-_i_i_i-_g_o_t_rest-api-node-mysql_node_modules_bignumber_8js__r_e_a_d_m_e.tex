

A Java\+Script library for arbitrary-\/precision decimal and non-\/decimal arithmetic.

\begin{DoxyItemize}
\item Integers and decimals
\item Simple A\+PI but full-\/featured
\item Faster, smaller, and perhaps easier to use than Java\+Script versions of Java\textquotesingle{}s Big\+Decimal
\item 8 KB minified and gzipped
\item Replicates the {\ttfamily to\+Exponential}, {\ttfamily to\+Fixed}, {\ttfamily to\+Precision} and {\ttfamily to\+String} methods of Java\+Script\textquotesingle{}s Number type
\item Includes a {\ttfamily to\+Fraction} and a correctly-\/rounded {\ttfamily square\+Root} method
\item Supports cryptographically-\/secure pseudo-\/random number generation
\item No dependencies
\item Wide platform compatibility\+: uses Java\+Script 1.\+5 (E\+C\+M\+A\+Script 3) features only
\item Comprehensive \href{http://mikemcl.github.io/bignumber.js/}{\texttt{ documentation}} and test set
\end{DoxyItemize}



If a smaller and simpler library is required see \href{https://github.com/MikeMcl/big.js/}{\texttt{ big.\+js}}. It\textquotesingle{}s less than half the size but only works with decimal numbers and only has half the methods. It also does not allow {\ttfamily NaN} or {\ttfamily Infinity}, or have the configuration options of this library.

See also \href{https://github.com/MikeMcl/decimal.js/}{\texttt{ decimal.\+js}}, which among other things adds support for non-\/integer powers, and performs all operations to a specified number of significant digits.\hypertarget{md__c_1__git_hub__p_r_o_y_e_c_t_o-_i_i_i-_g_o_t_rest-api-node-mysql_node_modules_bignumber_8js__r_e_a_d_m_e_autotoc_md3674}{}\doxysubsection{Load}\label{md__c_1__git_hub__p_r_o_y_e_c_t_o-_i_i_i-_g_o_t_rest-api-node-mysql_node_modules_bignumber_8js__r_e_a_d_m_e_autotoc_md3674}

The library exports a single constructor function, \href{http://mikemcl.github.io/bignumber.js/\#bignumber}{\texttt{ {\ttfamily Big\+Number}}}, which accepts a value of type Number, String or Big\+Number,


\begin{DoxyCode}{0}
\DoxyCodeLine{let x = new BigNumber(123.4567);}
\DoxyCodeLine{let y = BigNumber('123456.7e-\/3');}
\DoxyCodeLine{let z = new BigNumber(x);}
\DoxyCodeLine{x.isEqualTo(y) \&\& y.isEqualTo(z) \&\& x.isEqualTo(z);      // true}
\end{DoxyCode}


To get the string value of a Big\+Number use \href{http://mikemcl.github.io/bignumber.js/\#toS}{\texttt{ {\ttfamily to\+String()}}} or \href{http://mikemcl.github.io/bignumber.js/\#toFix}{\texttt{ {\ttfamily to\+Fixed()}}}. Using {\ttfamily to\+Fixed()} prevents exponential notation being returned, no matter how large or small the value.


\begin{DoxyCode}{0}
\DoxyCodeLine{let x = new BigNumber('1111222233334444555566');}
\DoxyCodeLine{x.toString();                       // "1.111222233334444555566e+21"}
\DoxyCodeLine{x.toFixed();                        // "1111222233334444555566"}
\end{DoxyCode}


If the limited precision of Number values is not well understood, it is recommended to create Big\+Numbers from String values rather than Number values to avoid a potential loss of precision.

{\itshape In all further examples below, {\ttfamily let}, semicolons and {\ttfamily to\+String} calls are not shown. If a commented-\/out value is in quotes it means {\ttfamily to\+String} has been called on the preceding expression.}


\begin{DoxyCode}{0}
\DoxyCodeLine{// Precision loss from using numeric literals with more than 15 significant digits.}
\DoxyCodeLine{new BigNumber(1.0000000000000001)         // '1'}
\DoxyCodeLine{new BigNumber(88259496234518.57)          // '88259496234518.56'}
\DoxyCodeLine{new BigNumber(99999999999999999999)       // '100000000000000000000'}
\DoxyCodeLine{}
\DoxyCodeLine{// Precision loss from using numeric literals outside the range of Number values.}
\DoxyCodeLine{new BigNumber(2e+308)                     // 'Infinity'}
\DoxyCodeLine{new BigNumber(1e-\/324)                     // '0'}
\DoxyCodeLine{}
\DoxyCodeLine{// Precision loss from the unexpected result of arithmetic with Number values.}
\DoxyCodeLine{new BigNumber(0.7 + 0.1)                  // '0.7999999999999999'}
\end{DoxyCode}


When creating a Big\+Number from a Number, note that a Big\+Number is created from a Number\textquotesingle{}s decimal {\ttfamily to\+String()} value not from its underlying binary value. If the latter is required, then pass the Number\textquotesingle{}s {\ttfamily to\+String(2)} value and specify base 2.


\begin{DoxyCode}{0}
\DoxyCodeLine{new BigNumber(Number.MAX\_VALUE.toString(2), 2)}
\end{DoxyCode}


Big\+Numbers can be created from values in bases from 2 to 36. See \href{http://mikemcl.github.io/bignumber.js/\#alphabet}{\texttt{ {\ttfamily A\+L\+P\+H\+A\+B\+ET}}} to extend this range.


\begin{DoxyCode}{0}
\DoxyCodeLine{a = new BigNumber(1011, 2)          // "11"}
\DoxyCodeLine{b = new BigNumber('zz.9', 36)       // "1295.25"}
\DoxyCodeLine{c = a.plus(b)                       // "1306.25"}
\end{DoxyCode}


Performance is better if base 10 is N\+OT specified for decimal values. Only specify base 10 when it is desired that the number of decimal places of the input value be limited to the current \href{http://mikemcl.github.io/bignumber.js/\#decimal-places}{\texttt{ {\ttfamily D\+E\+C\+I\+M\+A\+L\+\_\+\+P\+L\+A\+C\+ES}}} setting.

A Big\+Number is immutable in the sense that it is not changed by its methods.


\begin{DoxyCode}{0}
\DoxyCodeLine{0.3 -\/ 0.1                           // 0.19999999999999998}
\DoxyCodeLine{x = new BigNumber(0.3)}
\DoxyCodeLine{x.minus(0.1)                        // "0.2"}
\DoxyCodeLine{x                                   // "0.3"}
\end{DoxyCode}


The methods that return a Big\+Number can be chained.


\begin{DoxyCode}{0}
\DoxyCodeLine{x.dividedBy(y).plus(z).times(9)}
\DoxyCodeLine{x.times('1.23456780123456789e+9').plus(9876.5432321).dividedBy('4444562598.111772').integerValue()}
\end{DoxyCode}


Some of the longer method names have a shorter alias.


\begin{DoxyCode}{0}
\DoxyCodeLine{x.squareRoot().dividedBy(y).exponentiatedBy(3).isEqualTo(x.sqrt().div(y).pow(3))    // true}
\DoxyCodeLine{x.modulo(y).multipliedBy(z).eq(x.mod(y).times(z))                                   // true}
\end{DoxyCode}


As with Java\+Script\textquotesingle{}s Number type, there are \href{http://mikemcl.github.io/bignumber.js/\#toE}{\texttt{ {\ttfamily to\+Exponential}}}, \href{http://mikemcl.github.io/bignumber.js/\#toFix}{\texttt{ {\ttfamily to\+Fixed}}} and \href{http://mikemcl.github.io/bignumber.js/\#toP}{\texttt{ {\ttfamily to\+Precision}}} methods.


\begin{DoxyCode}{0}
\DoxyCodeLine{x = new BigNumber(255.5)}
\DoxyCodeLine{x.toExponential(5)                  // "2.55500e+2"}
\DoxyCodeLine{x.toFixed(5)                        // "255.50000"}
\DoxyCodeLine{x.toPrecision(5)                    // "255.50"}
\DoxyCodeLine{x.toNumber()                        //  255.5}
\end{DoxyCode}


A base can be specified for \href{http://mikemcl.github.io/bignumber.js/\#toS}{\texttt{ {\ttfamily to\+String}}}. Performance is better if base 10 is N\+OT specified, i.\+e. use {\ttfamily to\+String()} not {\ttfamily to\+String(10)}. Only specify base 10 when it is desired that the number of decimal places be limited to the current \href{http://mikemcl.github.io/bignumber.js/\#decimal-places}{\texttt{ {\ttfamily D\+E\+C\+I\+M\+A\+L\+\_\+\+P\+L\+A\+C\+ES}}} setting.


\begin{DoxyCode}{0}
\DoxyCodeLine{x.toString(16)                     // "ff.8"}
\end{DoxyCode}


There is a \href{http://mikemcl.github.io/bignumber.js/\#toFor}{\texttt{ {\ttfamily to\+Format}}} method which may be useful for internationalisation.


\begin{DoxyCode}{0}
\DoxyCodeLine{y = new BigNumber('1234567.898765')}
\DoxyCodeLine{y.toFormat(2)                       // "1,234,567.90"}
\end{DoxyCode}


The maximum number of decimal places of the result of an operation involving division (i.\+e. a division, square root, base conversion or negative power operation) is set using the {\ttfamily set} or {\ttfamily config} method of the {\ttfamily Big\+Number} constructor.

The other arithmetic operations always give the exact result.


\begin{DoxyCode}{0}
\DoxyCodeLine{BigNumber.set(\{ DECIMAL\_PLACES: 10, ROUNDING\_MODE: 4 \})}
\DoxyCodeLine{}
\DoxyCodeLine{x = new BigNumber(2)}
\DoxyCodeLine{y = new BigNumber(3)}
\DoxyCodeLine{z = x.dividedBy(y)                        // "0.6666666667"}
\DoxyCodeLine{z.squareRoot()                            // "0.8164965809"}
\DoxyCodeLine{z.exponentiatedBy(-\/3)                     // "3.3749999995"}
\DoxyCodeLine{z.toString(2)                             // "0.1010101011"}
\DoxyCodeLine{z.multipliedBy(z)                         // "0.44444444448888888889"}
\DoxyCodeLine{z.multipliedBy(z).decimalPlaces(10)       // "0.4444444445"}
\end{DoxyCode}


There is a \href{http://mikemcl.github.io/bignumber.js/\#toFr}{\texttt{ {\ttfamily to\+Fraction}}} method with an optional {\itshape maximum denominator} argument


\begin{DoxyCode}{0}
\DoxyCodeLine{y = new BigNumber(355)}
\DoxyCodeLine{pi = y.dividedBy(113)               // "3.1415929204"}
\DoxyCodeLine{pi.toFraction()                     // [ "7853982301", "2500000000" ]}
\DoxyCodeLine{pi.toFraction(1000)                 // [ "355", "113" ]}
\end{DoxyCode}


and \href{http://mikemcl.github.io/bignumber.js/\#isNaN}{\texttt{ {\ttfamily is\+NaN}}} and \href{http://mikemcl.github.io/bignumber.js/\#isF}{\texttt{ {\ttfamily is\+Finite}}} methods, as {\ttfamily NaN} and {\ttfamily Infinity} are valid {\ttfamily Big\+Number} values.


\begin{DoxyCode}{0}
\DoxyCodeLine{x = new BigNumber(NaN)                                           // "NaN"}
\DoxyCodeLine{y = new BigNumber(Infinity)                                      // "Infinity"}
\DoxyCodeLine{x.isNaN() \&\& !y.isNaN() \&\& !x.isFinite() \&\& !y.isFinite()        // true}
\end{DoxyCode}


The value of a Big\+Number is stored in a decimal floating point format in terms of a coefficient, exponent and sign.


\begin{DoxyCode}{0}
\DoxyCodeLine{x = new BigNumber(-\/123.456);}
\DoxyCodeLine{x.c                                 // [ 123, 45600000000000 ]  coefficient (i.e. significand)}
\DoxyCodeLine{x.e                                 // 2                        exponent}
\DoxyCodeLine{x.s                                 // -\/1                       sign}
\end{DoxyCode}


For advanced usage, multiple Big\+Number constructors can be created, each with their own independent configuration.


\begin{DoxyCode}{0}
\DoxyCodeLine{// Set DECIMAL\_PLACES for the original BigNumber constructor}
\DoxyCodeLine{BigNumber.set(\{ DECIMAL\_PLACES: 10 \})}
\DoxyCodeLine{}
\DoxyCodeLine{// Create another BigNumber constructor, optionally passing in a configuration object}
\DoxyCodeLine{BN = BigNumber.clone(\{ DECIMAL\_PLACES: 5 \})}
\DoxyCodeLine{}
\DoxyCodeLine{x = new BigNumber(1)}
\DoxyCodeLine{y = new BN(1)}
\DoxyCodeLine{}
\DoxyCodeLine{x.div(3)                            // '0.3333333333'}
\DoxyCodeLine{y.div(3)                            // '0.33333'}
\end{DoxyCode}
