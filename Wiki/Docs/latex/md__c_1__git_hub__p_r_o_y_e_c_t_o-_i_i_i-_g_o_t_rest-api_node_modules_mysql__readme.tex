If, for any reason, you would like to change the default connection flags, you can use the connection option {\ttfamily flags}. Pass a string with a comma separated list of items to add to the default flags. If you don\textquotesingle{}t want a default flag to be used prepend the flag with a minus sign. To add a flag that is not in the default list, just write the flag name, or prefix it with a plus (case insensitive).


\begin{DoxyCode}{0}

\DoxyCodeLine{  // disable FOUND\_ROWS flag, enable IGNORE\_SPACE flag}

\item {\ttfamily I\+G\+N\+O\+R\+E\+\_\+\+S\+I\+G\+P\+I\+PE} -\/ Don\textquotesingle{}t issue S\+I\+G\+P\+I\+PE if network failures. This flag has no effect on this Node.\+js implementation. (Default on)

\item {\ttfamily I\+N\+T\+E\+R\+A\+C\+T\+I\+VE} -\/ Indicates to the My\+S\+QL server this is an \char`\"{}interactive\char`\"{} client. This will use the interactive timeouts on the My\+S\+QL server and report as interactive in the process list. (Default off)
\item {\ttfamily L\+O\+C\+A\+L\+\_\+\+F\+I\+L\+ES} -\/ Can use {\ttfamily L\+O\+AD D\+A\+TA L\+O\+C\+AL}. This flag is controlled by the connection option {\ttfamily local\+Infile}. (Default on)

\item {\ttfamily M\+U\+L\+T\+I\+\_\+\+R\+E\+S\+U\+L\+TS} -\/ Can handle multiple resultsets for queries. (Default on)
\item {\ttfamily M\+U\+L\+T\+I\+\_\+\+S\+T\+A\+T\+E\+M\+E\+N\+TS} -\/ The client may send multiple statement per query or statement prepare (separated by {\ttfamily ;}). This flag is controlled by the connection option {\ttfamily multiple\+Statements}. (Default off)

\item Objects are turned into `key = \textquotesingle{}val'\`{} pairs for each enumerable property on the object. Most errors created by this module are instances of the Java\+Script \href{https://developer.mozilla.org/en/JavaScript/Reference/Global_Objects/Error}{\texttt{ Error}} object. Additionally they typically come with two extra properties\+:


\begin{DoxyItemize}
\item {\ttfamily err.\+code}\+: String, contains the My\+S\+QL server error symbol if the error is a \href{https://dev.mysql.com/doc/refman/5.5/en/server-error-reference.html}{\texttt{ My\+S\+QL server error}} (e.\+g. `\textquotesingle{}E\+R\+\_\+\+A\+C\+C\+E\+S\+S\+\_\+\+D\+E\+N\+I\+E\+D\+\_\+\+E\+R\+R\+OR'{\ttfamily ), a Node.\+js error code if it is a Node.\+js error (e.\+g.}\textquotesingle{}E\+C\+O\+N\+N\+R\+E\+F\+U\+S\+ED\textquotesingle{}{\ttfamily ), or an internal error code (e.\+g.}\textquotesingle{}P\+R\+O\+T\+O\+C\+O\+L\+\_\+\+C\+O\+N\+N\+E\+C\+T\+I\+O\+N\+\_\+\+L\+O\+ST\textquotesingle{}{\ttfamily ). $\ast$}err.\+errno\`{}\+: Number, contains the My\+S\+QL server error number. Only populated from \href{https://dev.mysql.com/doc/refman/5.5/en/server-error-reference.html}{\texttt{ My\+S\+QL server error}}.
\item {\ttfamily err.\+fatal}\+: Boolean, indicating if this error is terminal to the connection object. If the error is not from a My\+S\+QL protocol operation, this property will not be defined.
\item {\ttfamily err.\+sql}\+: String, contains the full S\+QL of the failed query. This can be useful when using a higher level interface like an O\+RM that is generating the queries.
\item {\ttfamily err.\+sql\+State}\+: String, contains the five-\/character S\+Q\+L\+S\+T\+A\+TE value. Only populated from \href{https://dev.mysql.com/doc/refman/5.5/en/server-error-reference.html}{\texttt{ My\+S\+QL server error}}.

You can also pass a function and handle type casting yourself. You\textquotesingle{}re given some column information like database, table and name and also type and length. If you just want to apply a custom type casting to a specific type you can do it and then fallback to the default.

The function is provided two arguments {\ttfamily field} and {\ttfamily next} and is expected to return the value for the given field by invoking the parser functions through the {\ttfamily field} object.

The {\ttfamily field} argument is a {\ttfamily Field} object and contains data about the field that need to be parsed. The following are some of the properties on a {\ttfamily Field} object\+:


\begin{DoxyItemize}
\item {\ttfamily db} -\/ a string of the database the field came from.
\item {\ttfamily table} -\/ a string of the table the field came from.
\item {\ttfamily name} -\/ a string of the field name.
\item {\ttfamily type} -\/ a string of the field type in all caps.
\item {\ttfamily length} -\/ a number of the field length, as given by the database.
\end{DoxyItemize}

The {\ttfamily next} argument is a {\ttfamily function} that, when called, will return the default type conversion for the given field.

When getting the field data, the following helper methods are present on the {\ttfamily field} object\+:


\begin{DoxyItemize}
\item {\ttfamily .string()} -\/ parse the field into a string.
\item {\ttfamily .buffer()} -\/ parse the field into a {\ttfamily Buffer}.
\item {\ttfamily .geometry()} -\/ parse the field as a geometry value.
\end{DoxyItemize}

The My\+S\+QL protocol is a text-\/based protocol. Based on the type information (like {\ttfamily I\+NT}), the type cast should convert the string field into a different Java\+Script type (like a {\ttfamily number}).
